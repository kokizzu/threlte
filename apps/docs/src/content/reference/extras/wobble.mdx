`<Wobble>` adds a time-varying vertex displacement to whatever material is already on its parent
mesh. Because it patches the material rather than replacing it, you can wobble any
`MeshStandardMaterial`, `MeshPhysicalMaterial`, `MeshLambertMaterial`, `MeshToonMaterial`, and so
on — pick the look you want, then drop `<Wobble>` in to make it move.

## Examples

### Basic Example

```svelte title="WobblingTorus.svelte"
<script lang="ts">
  import { T } from '@threlte/core'
  import { Wobble } from '@threlte/extras'
</script>

<T.Mesh>
  <T.TorusGeometry />
  <T.MeshStandardMaterial color="#ff6b6b" />
  <Wobble
    speed={2}
    factor={2}
  />
</T.Mesh>
```

The wobble is computed per-vertex, so meshes need enough geometric detail for the deformation
to look smooth. Increase the segment counts on the geometry if the surface looks faceted.

Setting `speed={0}` pauses the animation without unmounting the component.

### Anchoring the base

Pass `anchor` to pin a plane along the wobble axis. Amplitude grows with each vertex's distance
from that plane — vertices on the plane don't move, ones farther away move more. This is what
you want for a plant whose base shouldn't slide around in its pot:

```svelte title="AnchoredPlant.svelte"
<T.Mesh castShadow>
  <T is={plantGeometry} />
  <T.MeshStandardMaterial color="#4a7a3a" />
  <Wobble
    speed={1}
    factor={0.5}
    anchor={plantBaseY}
  />
</T.Mesh>
```

Shadows wobble alongside the visible mesh — `<Wobble>` attaches matching `customDepthMaterial`
and `customDistanceMaterial` to the parent so the shadow silhouette tracks the deformation.

### Pointing the bend

When `bendiness > 0`, an unset `forceDirection` produces a slowly drifting wind-like heading.
Set `forceDirection` when the force has a known direction — e.g. a blast from a specific side:

```svelte
<Wobble
  factor={1}
  bendiness={1}
  forceDirection={[1, 0, 0]}
/>
```

### Sideways objects

Default `axis` is `[0, 1, 0]` — Y-up. Pass a different axis for models whose "up" is along X or
Z (a vine, a horizontally-mounted flag, a Z-up model):

```svelte
<T.Mesh>
  <T is={vineGeometry} />
  <T.MeshStandardMaterial />
  <Wobble
    axis={[1, 0, 0]}
    bendiness={1}
    factor={0.4}
  />
</T.Mesh>
```

### Adjusting frequency for big or small geometry

`frequency` scales the per-vertex spatial sampling. The default (`1`) is tuned for ~1–3 unit
geometry; on much larger meshes the wobble looks uniform across the whole shape, on tiny ones
it gets too chaotic. Tune `frequency` to recover the look you want:

```svelte
<!-- Tall building-sized mesh -->
<Wobble
  factor={1}
  frequency={0.1}
/>
```

### Driving time externally

Provide `time` (in seconds) to drive the clock yourself. Useful for syncing many `<Wobble>`s to
a single timeline, scrubbing back and forth, or pausing:

```svelte
<script lang="ts">
  let clock = $state(0)
  // ...your own time source...
</script>

<T.Mesh>
  <T.SphereGeometry />
  <T.MeshStandardMaterial />
  <Wobble
    time={clock}
    factor={1}
  />
</T.Mesh>
```

### Reacting to material swaps

By default `<Wobble>` patches the parent mesh's material once when it mounts. If you swap
materials at runtime, pass it in via `material` so the patch follows:

```svelte
<script lang="ts">
  let { material } = $props()
</script>

<T.Mesh>
  <T.SphereGeometry />
  <T is={material} />
  <Wobble
    {material}
    speed={2}
    factor={1}
  />
</T.Mesh>
```
